## 核心功能拆解

### 1. 全自动流水线：30秒出片

```
用户输入 → LLM生成选题 → AI写脚本 → TTS配音
→ 图片素材抓取 → Ken Burns动画 → 字幕烧录
→ 多平台适配 → 输出发布包
```

整个流程无需人工干预，Ollama本地模型不够用时会自动降级到云端API（支持OpenAI/DeepSeek/通义千问）。

### 2. Ken Burns动画：让静态图动起来

这是这个项目的灵魂功能之一。

很多人以为做视频必须有素材、有拍摄、有剪辑基础。但Ken Burns效果告诉你：**一张静态图片，配合缓慢的缩放和平移，就能做出BBC纪录片的质感。**

```python
# 核心实现：FFmpeg zoompan滤镜
filter_str = (
    f"zoompan=z='if(lte(i,{frames}),"
    f"1.0+(1.3-1.0)*min(i/{frames},1),"
    f"1.3)':"
    f"x='iw/2-(iw/zoom/2)+{pan_x}':"
    f"y='ih/2-(ih/zoom/2)+{pan_y}':"
    f"d={frames}:s={width}x{height}:fps={fps}"
)
```

这个滤镜让图片从1.0x缓慢放大到1.3x，同时做水平/垂直平移。配合淡入淡出转场，4张图片就能拼接出一个45秒的完整视频。

### 4. 2D流程图动画：架构讲解神器

这是最让我兴奋的功能。

很多技术内容需要展示系统架构，比如"用户请求是怎么被处理的"、"微服务之间怎么通信"。

传统做法是用PPT画好，上传到视频编辑软件加动画。

现在，你只需要在脚本里写一个代码块：

````
```diagram
[client] 用户浏览器 (300, 80, 160, 70, teal)
[cdn] CDN节点 (150, 200, 160, 70, blue)
[server] API网关 (450, 200, 160, 70, blue)
[auth] 认证服务 (150, 350, 160, 70, orange)
[db] 数据库 (450, 350, 160, 70, orange)

[client] -> [cdn] DNS解析
[cdn] -> [server] 回源
[server] -> [auth] JWT验证
```
````

系统自动解析这段DSL，生成矩形模块、箭头连线、逐帧绘制动画。元素按顺序出现，箭头从无到有生长，完全模拟人工绘制过程。

### 5. 多平台智能适配：一条视频，多种吃法

同一套素材，发布到不同平台需要不同的处理：

| 平台 | 标题风格 | 时长建议 | 封面文字 |
|------|---------|---------|---------|
| 抖音 | 悬念式、数字钩子 | 15-60秒 | 少字、大字 |
| 小红书 | 干货感、情绪感 | 10秒-5分钟 | 多字、精致 |
| B站 | 系列感、深度感 | 30秒-10分钟 | 适中、可系列 |

系统会自动生成各平台的**标题、描述、话题标签、封面建议**，并打包成独立的文件夹。

## 技术选型：为什么这样设计

### TTS配音降级链

```
讯飞TTS（付费）→ 百度TTS（付费）→ Edge-TTS（免费）→ gTTS（免费）→ SAPI（系统自带）
```

为什么要做5层降级？因为每个TTS服务都有各自的稳定性问题。讯飞可能配额用完，百度可能key过期，Edge-TTS在国内可能访问不稳……

但只要有一层能用，视频就能生成。**这是真正的生产级设计。**

### 图片抓取降级链

```
Pexels API（每天200张免费）→ Unsplash API（每月50次）→ Bing图片爬虫（无限制）
```

Pexels和Unsplash都需要申请API Key。对于不想折腾的用户，Bing图片搜索爬虫提供了完全无门槛的降级方案——直接爬Bing的图片结果页，无需任何Key。

### 视频处理：为什么是FFmpeg？

因为它是唯一一个在所有平台上都稳定可用的视频处理工具。

Python生态里有moviepy、opencv、pillow……但它们的跨平台兼容性、视频编码质量、生产稳定性都比不上FFmpeg。

**FFmpeg是视频处理领域的C语言**：上古神器，稳定可靠，参数即API。

## 实际使用效果

我拿这个工具做了三轮测试：

**第一轮：知识付费类（职场晋升）**
- 选题方向："汇报工作记住这三点"
- 素材：4张职场场景图
- 输出：抖音发布包（标题：领导听完我的汇报，只说了三个字）
- 平台：B站
- 耗时：38秒

**第二轮：科技数码类（Python异步编程）**
- 选题方向："asyncio三分钟入门"
- 素材：自动抓取（关键词：Python, async）
- 输出：技术讲座风格视频（Ken Burns + 代码高亮）
- 平台：B站 + 知乎
- 耗时：52秒

**第三轮：生活类（极简生活）**
- 选题方向："女生极简生活的5个阶段"
- 素材：用户上传（10张图片）
- 输出：Mode B模式（素材拼接 + BGM）
- 平台：抖音
- 耗时：25秒

三轮测试全部成功出片。最让我惊讶的是第三轮——用户只需要提供图片素材，整个流程就自动跑完了。

## 这个项目适合谁

**✅ 非常适合：**
- 有一定技术背景的创作者（会写代码、调API）
- 技术类/知识类UP主（编程、教学、数码评测）
- 想做矩阵号但人力有限的团队
- 对视频质量有要求、不想用PPT式简陋效果的人

**❌ 不适合：**
- 完全不懂技术的小白（需要配置Python环境、FFmpeg）
- 需要真人出镜的口播内容（目前只能做图文类）
- 对动画复杂度要求极高的（比如逐帧手绘动画）
